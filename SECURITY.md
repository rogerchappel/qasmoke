# Security Policy

## Supported Versions

This is an early MVP. Security fixes land on the latest `main` branch.

## Reporting a Vulnerability

Please do not open a public issue for a suspected vulnerability.

Instead, email the maintainer or use GitHub private vulnerability reporting if enabled. Include:

- affected version or commit
- reproduction steps
- impact assessment
- any suggested mitigation

## Security stance

qasmoke is intentionally local-first:

- no hidden network calls
- no telemetry
- no background secret discovery
- no credential collection
